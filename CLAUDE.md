# ReBalance — Unraid Array Disk Usage Balancer

Custom Unraid plugin that moves files between array disks to equalise usage
percentage across mixed-capacity drives (e.g. 3.6 TB and 1.8 TB drives in the
same array). Targets a weighted average usage % so every disk lands at
roughly the same fill level rather than the same raw byte count.

## How to build and deploy

Build requires Linux/WSL (uses `tar` with `--transform`).
ImageMagick must be installed in WSL to generate the PNG icon:
```bash
sudo apt-get install imagemagick
```

```bash
# From WSL:
cd /mnt/c/Users/steve/OneDrive/Documents/Scripts/ReBalance
bash package.sh
```

`package.sh` does two things:
1. Converts `plugin/images/rebalance.svg` → `plugin/images/rebalance.png` via ImageMagick
   (Unraid's Tools menu requires a raster image; SVG is not rendered in `<img>` tags)
2. Builds `rebalance.tar.gz`

**For a new release, also:**
1. Update the version entity in `rebalance.plg` (format: `YYYY.MM.DD`)
2. Add changelog entry to `<CHANGES>` in `rebalance.plg`
3. Recalculate the MD5: `md5sum rebalance.tar.gz`
4. Update `<MD5>` in `rebalance.plg`
5. Commit and push
6. Create a new GitHub Release tagged with the version and attach `rebalance.tar.gz`
7. Update the `<URL>` in `rebalance.plg` to point to the new release download

**For local testing (SSH to Unraid):**
```bash
mkdir -p /boot/config/plugins/rebalance
cp rebalance.plg     /boot/config/plugins/rebalance/
cp rebalance.tar.gz  /boot/config/plugins/rebalance/
```
Then in Unraid web UI: **Plugins → Install Plugin**, browse to
`/boot/config/plugins/rebalance/rebalance.plg`.

Note: Unraid's Community Applications pre-hook logs
`"Cannot install rebalance.tar.gz: invalid package extension"` — harmless.

## Architecture

### rebalance.sh
Pure bash, runs as a background daemon launched by PHP.

Key flow:
1. Writes PID to `/tmp/rebalance.lock` (singleton guard)
2. Writes initial `state=starting` to `/tmp/rebalance_status.json` immediately
   (so the PHP poller never sees a "running but no status" window)
3. `discover_disks` — scans `/mnt/disk[0-9]*`, reads `df` stats
4. `calc_target` — computes weighted average usage % across all disks
5. `build_plan` — identifies source disks (above target+tolerance), scans
   files, assigns each file to the best destination disk, updates virtual
   disk state as files are planned. Emits `planning_disk_index` /
   `planning_disk_count` so the UI can show a deterministic progress bar.
6. If `--dry-run`: exits with `STATE=planned`, no files moved
7. If nothing to do: exits with `STATE=completed`
8. `execute_plan` / `execute_plan_cached` — moves files via `rsync -aX`,
   updates stats, writes status every 5 seconds
9. `_on_exit` trap — sets final state, writes status, cleans staging dir,
   removes lock file

Status is written atomically via `mktemp` + `mv` to avoid partial reads.

**Cache-assisted mode** (`--use-cache`): while file N is being rsync'd to its
destination (slow parity write), file N+1 is pre-staged from source to the
SSD cache in the background. Gives ~1.5× throughput by overlapping source
reads with destination writes on separate physical devices.

**Min file size filter** (`--min-file-kb N`): `find` uses `-size +Nk` to skip
small files (subtitles, NFOs, thumbnails). `size_opt` must be set BEFORE the
`while` loop — bash evaluates the process substitution `< <(find ...)` at
loop setup time, before any iteration runs.

### status.php
Thin PHP AJAX endpoint. No framework.

- **GET** `/plugins/rebalance/scripts/status.php?tolerance=N`
  - If script running + status file exists → return status file verbatim
  - If script running + no status file yet → return synthetic `state=starting`
  - If not running → return `idle_payload()` overlaying last run's results
  - Response always includes `cache_pools` array (detected from `disks.ini`)
- **POST** actions: `start`, `stop`, `clear`
  - All POST requests **must include `csrf_token`** — Unraid's emhttp silently
    drops POSTs without it (returns HTTP 200 with empty body)
  - `launch_script()` uses `exec()` with `nohup ... </dev/null >/dev/null 2>&1 &`
  - `start` accepts: `tolerance`, `dry_run`, `use_cache`, `cache_buffer_kb`,
    `stage_dir`, `min_file_kb`

Key defensive patterns:
- `ob_start()` at top captures stray output; `ob_clean()` in `json_out()` before every echo
- `catch (Throwable $e)` wrapper ensures all PHP errors return valid JSON

### ReBalance.page
Unraid `.page` file format: PHP frontmatter + HTML/CSS/JS in one file.

Frontmatter:
```
Menu="Utilities"
Title="ReBalance"
Tag="balance-scale"
Icon="images/rebalance.png"
```
- `Tag` — drives the scales icon in the plugin page header (Font Awesome)
- `Icon` — drives the PNG shown in the Tools menu tile (must be raster, not SVG)
  Both fields are needed; removing either breaks one of the two icon locations.

PHP reads CSRF token: `$var = @parse_ini_file('/var/local/emhttp/var.ini') ?: [];`
Token injected into JS as `var CSRF_TOKEN = '...'` and sent with every POST.

JS polling: `setInterval(poll, 2000)` while `running/planning/starting`.
Stops automatically on terminal state (`completed/planned/stopped/error`).

**Inline progress bar** (right of Stop button):
- `starting` → animated blue sweep (indeterminate)
- `planning` (before first disk) → animated blue sweep
- `planning` (once `planning_disk_index` > 0) → deterministic solid blue bar,
  label: `Disk 3 of 6  —  593,484 files / 748.9 GB planned  (~12 min left)…`
  ETA = `elapsed ÷ disks_done × disks_remaining` (shown after 2+ disks done)
- `running` → solid green, fills to `stats.pct_done` %
- `completed` → full green, "100%"
- `planned` → full green, "Ready"
- `stopped` → orange, frozen at % reached
- `error` → full red, "Error"

**Buttons:**
- "Start Analysis" — always forces `dry_run=true`; never moves files
- "Start Rebalance" — live run, moves files

**Dry Run checkbox removed** — Start Analysis always performs a dry run, so
the checkbox was redundant and potentially confusing.

**Minimum File Size filter** — number field (MB) on same line as Dark Mode
toggle. Default 0 (no filter). Sent as `min_file_kb` (MB × 1024) to script.

**Cache pool dropdown** — auto-populated from `cache_pools` in GET response.

## Key bugs fixed during development

### 1. CSRF token (root cause of all early POST failures)
Unraid's emhttp validates `csrf_token` on all POSTs. Missing → HTTP 200 empty body.
Syslog: `webGUI: E plugins/rebalance/scripts/status.php - missing csrf_token`.
Fix: read from `/var/local/emhttp/var.ini`, inject into JS, send with every POST.

### 2. proc_open blocks PHP response
`proc_open()` + `proc_close()` waits for child to exit — holds HTTP connection
open for multi-hour runs. Fix: `exec('nohup cmd </dev/null >/dev/null 2>&1 &')`.

### 3. Poller race condition (script alive but no status file yet)
Lock file written → status file not yet written → PHP returns `idle` → JS stops
polling → results never displayed. Fix: early `write_status` in bash after lock
write; PHP returns synthetic `state=starting` if running but no status file.

### 4. Empty JSON responses (ob_start / stray output)
PHP output before `echo json_encode()` corrupts JSON.
Fix: `ob_start()` at file top, `ob_clean()` inside `json_out()`.

### 5. Wrong staging directory (hardcoded /mnt/cache)
Staging hardcoded to `/mnt/cache/.rebalance_stage`. User's pool is `ssd_cache`
at `/mnt/ssd_cache`. Fix: `detect_cache_pools()` reads `disks.ini`; bash does
same with `awk`. `--stage-dir` override also supported.

### 6. Planning took hours (find | sort pipeline)
`find … | sort -k1,1rn` buffered entire file list before processing.
Fix: removed `sort` entirely — files processed in find's natural order.

### 7. Min file size filter never applied (size_opt placement)
`local size_opt=""` and assignment were inside the `while` loop body before
`done < <(find ... ${size_opt} ...)`. Bash expands `${size_opt}` at loop setup
time (before first iteration) so it was always empty.
Fix: moved both lines to before the `while` statement.

### 8. Duplicate START_TIME declaration
`START_TIME=$(date +%s)` accidentally duplicated during bug #3 fix, resetting
elapsed time mid-run. Fix: removed duplicate.

### 9. Tools menu icon not showing (SVG in img tag)
`Tag="balance-scale"` showed generic page icon (FA class not found in Unraid's
FA version). Switching to `Icon="images/rebalance.svg"` showed nothing — Unraid's
Tools page doesn't render SVG in `<img>` tags. Fix: `package.sh` converts SVG
→ PNG via ImageMagick; `Icon="images/rebalance.png"` references the PNG.
`Tag="balance-scale"` kept for the in-page header icon (uses Font Awesome
directly in HTML, which does work). Both fields must be present.

## Status JSON fields (written by rebalance.sh)

| Field | Type | Notes |
|---|---|---|
| `state` | string | `starting/planning/running/planned/completed/stopped/error` |
| `dry_run` | bool | true if `--dry-run` was passed |
| `use_cache` | bool | true if cache-assisted mode |
| `tolerance` | int | ±% tolerance around target |
| `target_pct` | int | weighted average usage % |
| `elapsed_sec` | int | seconds since script started |
| `planning_disk_index` | int | current source disk being scanned (1-based) |
| `planning_disk_count` | int | total source disks to scan |
| `plan_count` | int | file moves planned so far |
| `stats.files_total` | int | same as plan_count after planning |
| `stats.bytes_total_kb` | int | total KB to move |
| `stats.pct_done` | int | execution progress % |
| `disks` | array | per-disk name/size/used/free/pct/role |
| `plan` | array | first 300 planned moves (status/kb/src/dst) |
| `log` | array | last 100 log entries ({t, m}) |
| `cache_pools` | array | idle GET only; detected cache pools |

## Unraid-specific notes

- **Plugin files** land in `/usr/local/emhttp/plugins/rebalance/` — wiped on
  every reinstall, not persisted across reboots. Runtime state (`/tmp/`) also
  lost on reboot.
- **Boot persistence**: `.plg` and `.tar.gz` on USB boot drive at
  `/boot/config/plugins/rebalance/`. Reinstalled on every boot from there.
- **PHP environment**: lighttpd + PHP-FPM. `exec()` and `shell_exec()` available.
  `proc_open` available but blocks on `proc_close`.
- **Disk paths**: array disks at `/mnt/disk1`, `/mnt/disk2`, etc.
  Cache pool at `/mnt/ssd_cache` (detected at runtime from `disks.ini`).
- **File system**: XFS on all array disks. `rsync -aX` preserves extended attrs.
- **Parity**: updated on every write. File moves trigger parity writes on both
  source and destination — normal and expected.
- **Page menu**: `Menu="Utilities"` puts plugin under **Tools** in Unraid nav.
- **Background process**: runs as a fully detached daemon — survives browser
  tab closes, page navigation, and PHP connection drops.
